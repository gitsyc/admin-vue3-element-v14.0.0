# 车企销售运营平台设计方案

## 一、项目概述

### 1.1 背景
主机厂需要统一管理区域、经销商与门店经营过程，打通从获客到成交再到回款、交付与车主运营的全链路，同时实现组织权限与数据口径统一，支撑总部经营监控与精细化运营。

### 1.2 目标
- 打通业务闭环：线索 → 跟进 → 试驾 → 报价 → 订单 → 收款 → 交付 → 车主运营
- 统一协同体系：总部/区域/经销商/门店/员工多级组织与权限、流程与数据口径一致
- 建立经营能力：总部驾驶舱、报表中心与预警体系，支持下钻分析与执行闭环

### 1.3 核心用户与角色
- 总部：销售运营、渠道管理、库存/配车、财务管理、市场/用户运营、数据分析
- 区域：区域运营、区域财务、区域渠道管理
- 经销商/门店：店长、销售经理、销售顾问、展厅/交付、财务出纳、市场专员

## 二、功能范围

### 2.1 组织与权限中心（底座）
- 组织架构：总部 → 区域 → 经销商 → 门店 → 员工
- 权限体系：菜单/按钮权限 + 数据权限（区域/门店/个人）+ 关键业务权限（价格、作废、调拨等）
- 审计留痕：关键单据与财务操作全量记录（操作人、时间、字段变更）

### 2.2 线索运营中心
- 线索接入：导入/接口/渠道来源管理
- 分配与协同：自动分配、手动转派、共享池/保护期（可配置）
- 跟进与转化：跟进记录、预约到店、试驾管理、战败原因结构化
- 转化分析：漏斗、渠道质量、门店/个人绩效

### 2.3 整车采购与库存协同
- 采购计划：总部计划、区域配额、门店申报与汇总审核
- 车辆资源池：车型/配置/颜色/VIN、在途/入库/可售等状态
- 配车与调拨：锁车、调拨、退库/回收（可选）与审批流

### 2.4 整车销售中心
- 报价与政策：报价单、销售政策应用、底价/折扣权限控制
- 订单与合同：订单创建、审批、合同信息、交付计划
- 交付协同：交付清单、车辆状态核验、交付完成闭环

### 2.5 财务管理中心（一期够用）
- 资金：收款/退款、应收台账、开票状态
- 费用：活动费用/门店费用台账、返利/补贴核算（基础版）
- 规则：回款未完成不可交付；退款需审批并联动订单与库存状态

### 2.6 用户运营中心（一期基础版）
- 客户主档：潜客/车主统一档案、标签画像、去重与合并策略
- 运营动作：回访、活动触达、复购/转介绍登记与效果追踪

### 2.7 报表中心与总部驾驶舱
- 指标体系：统一口径（线索、成交、库存周转、回款达成等）
- 看板与下钻：总部/区域/门店/个人多层级展示与明细下钻
- 异常预警：线索超时、库存积压、回款异常、门店转化异常

### 2.8 消息与待办中心
- 待办：审批、超时跟进、库存/经营预警、交付提醒
- 通知：政策下发、活动通知、任务下发（可选）

## 三、核心模块设计（业务域与关键规则）

### 3.1 组织与权限底座
- 数据隔离默认原则：经销商间不可互看；门店仅看本店；个人仅看本人客户/单据（可按角色放开）
- 关键业务权限：低价销售审批、订单作废/反作废、调拨审批、退款审批
- 组织调整：门店划转时需定义历史数据归属与报表统计口径（按发生时组织/按当前组织，二选一配置）

### 3.2 线索到成交闭环
- 重复识别：手机号/证件号/微信等命中时提示合并或转共享池
- 保护期：线索分配后在保护期内仅负责人可操作（可配置）
- 超时规则：未首次跟进/未阶段推进触发提醒与升级（销售经理/店长/区域）
- 战败原因：必填结构化字段，支持总部复盘渠道质量与话术/产品问题归因

### 3.3 采购、库存与订单联动
- 车辆状态建议：待采购/在途/已入库/可售/已锁定/已销售/已交付
- 锁车规则：订单创建可锁定库存车；订单取消/改配需释放锁定并记录原因
- 调拨规则：跨门店/跨经销商调拨必须审批；调拨影响库存与可售状态

### 3.4 销售、交付与财务联动
- 价格控制：低于底价或低于政策价触发审批；审批通过后才能生效
- 交付核验：车辆状态、订单状态、收款状态、发票状态满足条件才允许交付完成
- 退款回滚：退款后自动联动订单/财务/库存状态，保留完整操作链路

### 3.5 用户运营生命周期
- 生命周期：潜客（线索/跟进）→ 成交客户（订单）→ 车主（交付后）→ 复购/转介绍
- 效果追踪：活动触达 → 到店/试驾/成交的转化归因（基础版先做关联与统计）

## 四、数据结构与流向（概要）

### 4.1 关键实体（主数据与业务单据）
- 组织：Org（总部/区域/经销商/门店）、Employee、Role、Permission、DataScope
- 客户：Customer（潜客/车主）、Contact、Tag、CustomerMergeLog
- 线索：Lead、LeadAssignment、FollowUp、TestDrive、LoseReason
- 车辆：Model、Config、Vehicle（VIN）、Inventory、Transfer（调拨）
- 销售：Quote、Order、Contract、Delivery
- 财务：Receipt、Refund、Invoice、Expense、Subsidy（基础）
- 运营：Campaign、TouchRecord、Referral、Member（可选）

### 4.2 核心数据流
- 获客链路：Lead → FollowUp/TestDrive → Quote → Order
- 交易链路：Order → Receipt/Invoice → Delivery → Customer（车主）
- 供应链路：Plan/Allocation → Vehicle(InTransit) → Inventory(InStore) → Lock → Sold/Delivered
- 分析链路：各域事件/单据 → 指标口径层 → 报表中心/驾驶舱 → 预警与待办

## 五、边界情况处理（一期必须覆盖）
- 重复线索/重复客户：去重、合并、归属与保护期冲突处理
- 抢客与协同：共享池、转派审批（可选）、跨门店协同规则
- 无库存成交：预售/申请配车/调拨流程与客户承诺管理
- 订单变更：改配、取消、退订的库存释放与财务回滚
- 数据隔离：经销商隔离、区域下钻、总部全局视角并存
- 组织变更：门店划转后的统计口径与历史数据可追溯
- 异常预警：线索超时、库存周转异常、回款异常、门店转化异常

## 六、一期交付建议（全链路但分深浅）
- 做深：组织权限、线索、销售订单、库存协同、经营看板与预警
- 做够用：财务台账（收退款/应收/开票）、用户运营基础（档案/标签/回访/活动）、报表中心（统一口径）
- 预留扩展：营销自动化、会员积分完善、完整财务/ERP对接、呼叫中心/企微集成、AI 线索评分与跟进建议
